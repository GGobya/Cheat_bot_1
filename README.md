# Behavior Cheat Engine — Telegram-бот

Бот выдаёт по одному поведенческому «читу» в день: короткое объяснение + микродействие
+ кнопки «Сделал / Не сделал». При «не сделал» — предлагает упрощённую версию (10%).

## Структура

- `main.py` — точка входа: хендлеры aiogram + планировщик ежедневной рассылки
- `config.py` — настройки (токен берётся из переменной окружения `BOT_TOKEN`)
- `db.py` — слой работы с SQLite (пользователи, ответы, прогресс)
- `cheats.py` — контент: 30 читов
- `messages.py` — тексты сообщений и инлайн-клавиатуры
- `requirements.txt` — зависимости
- `Procfile` — команда запуска для хостингов, которые его читают

## Запуск на хостинге (bothost и подобные)

1. Залей этот репозиторий на GitHub (см. команды ниже).
2. В панели хостинга подключи репозиторий.
3. Укажи команду запуска: `python main.py` (или хостинг определит её сам по `Procfile`).
4. В разделе переменных окружения (Environment Variables / Variables) добавь:
   - `BOT_TOKEN` = токен, который дал @BotFather
5. Запусти бота (Start/Deploy).

**Важно про базу данных:** бот хранит прогресс в файле `bcheat.db` (SQLite) рядом со
скриптом. На некоторых хостингах файловая система очищается при каждом передеплое —
тогда прогресс пользователей будет сбрасываться. Если хостинг это поддерживает,
подключи постоянный диск (persistent volume/storage) и положи туда базу, либо
переключись на внешнюю БД (Postgres) при необходимости — для этого нужно будет
заменить `aiosqlite`-запросы в `db.py`.

## Локальный запуск (для проверки перед деплоем)

```bash
pip install -r requirements.txt
export BOT_TOKEN="твой_токен_от_BotFather"
python main.py
```

## Команды бота

- `/start` — войти в эксперимент, сразу получить День 1
- `/settime 9` — во сколько часов (0–23, по времени сервера) приходит ежедневный чит
- `/status` — прогресс: текущий день, выполнено/пропущено, серия подряд
- `/stop` — поставить рассылку на паузу
